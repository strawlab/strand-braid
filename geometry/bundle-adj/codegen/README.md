# Bundle-adjustment symbolic Jacobian codegen

This directory contains the [SymPy](https://www.sympy.org/) scripts that
derive the analytical Jacobians used by the `bundle-adj` crate.

The codegen is **not** part of a normal `cargo build`. It produces complete,
ready-to-use Rust modules in `generated/` that are included via `include!()` in
`../src/lib.rs`.

Running the codegen is only needed when the symbolic model changes (i.e.
approximately never). The codegen handles all formatting, header removal, and
rustfmt guards.

The main scripts are:

* `regenerate.sh` — the supported end-to-end workflow.
* `extrinsics-only.py`, `opencv5.py`, `opencv4.py`, and `opencv0.py` — model-specific symbolic Jacobian generators.
* `camera_codegen.py` — shared symbolic helpers used by the generators and validation.
* `validate_jacobian.py` — finite-difference sanity check for the extrinsics-only symbolic Jacobian.
* `build_modules.py` — assembles the final Rust modules in `generated/` from the intermediate `scratch/` outputs.

## How to regenerate

Prerequisite: [`uv`](https://docs.astral.sh/uv/). The codegen project is pinned
to Python 3.14 in `pyproject.toml`, and `uv run --project . ...` will create/use
the local `.venv/` automatically. Install `uv` with:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Then run:

```bash
./regenerate.sh
```

This runs the model generator scripts, validates the extrinsics-only symbolic
Jacobian against finite differences, and then assembles the final Rust modules
from the intermediate `scratch/*_{cam,pt}.rs` files.

This produces:

* `generated/extrinsics_only.rs` — the Jacobian functions for `CameraModelType::ExtrinsicsOnly`.
* `generated/opencv5.rs` — the Jacobian functions for `CameraModelType::OpenCV5`
  (focal length `f`, 3 radial + 2 tangential distortion coefficients, 3-axis
  Rodrigues rotation and 3-axis world translation). Also contains the shared
  point jacobian function for all OpenCV-style intrinsic models.
* `generated/opencv4.rs` — the Jacobian functions for `CameraModelType::OpenCV4`
  (focal length `f`, 2 radial + 2 tangential distortion coefficients).
* `generated/opencv0.rs` — the Jacobian functions for `CameraModelType::OpenCV0`
  (focal length `f` only, no distortion coefficients).

To verify that the regeneration is deterministic and the generated files in
`generated/` are up to date, run:

```bash
./regenerate.sh --check
```

which additionally fails if `git diff` reports any change under `generated/`.

## How the generated code maps into the Rust source

The codegen produces complete Rust modules with functions that return Jacobian
values. These modules are included via `include!()` macros:

| Generated module                      | Function(s) exported                                      |
| ------------------------------------ | --------------------------------------------------------- |
| `generated/opencv5.rs`              | `opencv5_cam_jacobian`, `opencv_intrinsics_point_jacobian` |
| `generated/opencv4.rs`               | `opencv4_cam_jacobian`                                    |
| `generated/opencv0.rs`               | `opencv0_cam_jacobian`                                    |
| `generated/extrinsics_only.rs`       | `extrinsics_only_cam_jacobian`, `extrinsics_only_point_jacobian` |

All formatting, header removal, rustfmt guards, and indentation is handled
automatically by `build_modules.py`. No manual editing is required.

### OpenCV-style intrinsic models: `OpenCV5`, `OpenCV4`, and `OpenCV0`

`camera_models.rs` contains three `CameraModelType` variants for OpenCV-style
intrinsic models. For their pinhole model, these have focal length `f` and
center `(cx, cy)`. (The horizontal and vertical focal length are equal and skew
is zero.) `OpenCV5` has 5 distortion coefficients, `OpenCV4` has 4, and
`OpenCV0` has 0 (no distortion). These are produced by the codegen scripts:
`opencv5.py`, `opencv4.py`, and `opencv0.py`.

All OpenCV-style intrinsic models share the same point jacobian structure,
which is generated by `opencv5.py` and exported as
`opencv_intrinsics_point_jacobian`. The only difference between models is
which distortion parameters are non-zero (all 5 for OpenCV5, first 4 for
OpenCV4, none for OpenCV0), but the jacobian structure is the same.

## Validation

`validate_jacobian.py` sanity-checks the symbolic Jacobian for the
extrinsics-only model against a finite-difference estimate of the residuals.
It reuses the shared symbolic model builders from `camera_codegen.py`, so the
validation path and generator path stay aligned. It is invoked automatically by
`regenerate.sh`.
